* A questão de não ser um servidor 
  qdo é servidor vc tem um processo que intermedia o seu programa que
  acessa o banco e o banco propriamente dito.
  No caso so sqlite, o processo do seu aplicativo acessa o banco e é o
  seu processo mesmo que escreve no arquivo de banco de dados sqlite
  no disco.

  Dos que bancos que funcionam desse jeito o sqlite é o único que
  permite vários processos acessarem o banco ao mesmo tempo.
